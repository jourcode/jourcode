# Security Policy

This repository powers the public JourCode GitHub profile.

## Scope

Security concerns in scope include:

- exposed secrets or private identifiers in public profile content
- unsafe security guidance in public-facing text
- broken public links that could mislead visitors
- accessibility barriers in profile material that block access to security
  information

Private account details, raw evidence, personal records, and unrelated systems
are out of scope for this repository.

## Reporting

Use a GitHub issue for public-safe profile concerns when available.

Do not include live passwords, tokens, recovery codes, private messages, account
records, screenshots with personal data, or sensitive evidence in public issues.

If a concern involves private material, keep the report minimal and ask for a
private path before sharing details.

## Standard

Security should protect people without taking away their access, privacy, or
autonomy. Profile content should reflect that standard without exposing private
operational details.
